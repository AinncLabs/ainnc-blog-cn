# 手把手教程:3 步搭建你的第一个自动化养号任务

这篇是纯操作向的教程,目标是让第一次接触批量运营工具的人,能在 10 分钟内跑起第一个养号任务。

## 准备工作:先把账号和设备配好

在发布任何任务之前,账号必须先绑定设备。这是最容易被新手跳过、但最重要的一步——没有独立设备绑定的账号,后续所有操作都在裸奔状态下进行。

进入「我的账号」→「创建」,批量导入账号信息,系统会为每个账号自动分配一台独立云手机:

| 字段 | 说明 |
|---|---|
| 平台 | TikTok / Instagram / Reddit / X |
| 用户名 | 账号登录名 |
| 邮箱 | 用于登录验证的邮箱 |
| 分组 | 建议按项目或客户命名,例如 `TK-新品预热组` |
| 绑定设备 | 系统自动分配,也可手动指定 |

**建议**:导入时就把分组名规划好,比事后补分组省事得多。比如按"平台+用途"命名:`TK-养号中`、`IG-已可发布`。

## 第一步:进入任务中心,选择「养号」任务

任务中心按平台和类型筛选,选中对应平台的「养号」任务卡片。养号任务的核心是配置一段拟人化行为序列,常见的参数结构是这样的:

```json
{
  "task_type": "养号",
  "platform": "TikTok",
  "account_group": "TK-新品预热组",
  "behavior_sequence": [
    { "action": "浏览", "duration_min": 5, "duration_max": 12 },
    { "action": "点赞", "count_min": 3, "count_max": 8 },
    { "action": "关注", "count_min": 0, "count_max": 2 },
    { "action": "评论", "count_min": 0, "count_max": 1 }
  ],
  "run_window": "09:00-11:00",
  "randomize_offset_min": 20
}
```

几个新手容易踩的坑:

- **关注数量别贪多**:新号第一周关注量控制在个位数,尤其是 Instagram,过量关注是触发风控的头号信号。
- **别用固定时间点**:`run_window` 配合随机偏移(`randomize_offset_min`),避免所有账号在同一秒发起请求。
- **行为序列要有停留时长**:纯"秒赞"没有停留时长的行为,平台很容易识别为脚本操作。

## 第二步:选择账号

按分组选择要执行任务的账号,支持批量勾选。第一次跑建议先选 5–10 个账号做小范围验证,观察一天的执行结果,确认稳定后再放量到整个分组。

## 第三步:填写参数并确认发布

检查行为序列参数、执行时间窗口,确认无误后提交。提交后可以在「我的任务」里实时查看每个账号的执行状态:

```
任务ID  账号            状态      执行情况
2106    charlesmccoywr704   成功    执行完成
2105    briangouldef099     等待中   启动中
2104    victoriacoxwb927    等待中   启动中
```

系统每 10 秒自动刷新状态,失败的任务点击「详情」和「日志」可以看到具体失败原因,方便针对性调整参数,而不是盲目重试。

## 小范围验证之后,再谈规模化

第一批任务跑完观察 24–48 小时,重点看两个数字:登录状态是否稳定、任务成功率是否达到预期(建议不低于 85%)。达标之后,再把同样的参数模板套用到整个分组,这比一上来就对几百个账号同时操作稳妥得多。

## 写在最后

自动化工具能解决"重复劳动"的问题,但参数设计本身依然需要运营经验的判断——工具负责执行,人负责判断"这套行为序列像不像真人"。把这一步做扎实,后面的规模化才不会踩坑。
